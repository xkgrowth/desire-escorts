# 2. Core Philosophy (Аксиома системы)
В Strapi v5 произошли радикальные архитектурные изменения по сравнению со Strapi v4. 
СТРОЖАЙШИЙ ЗАПРЕТ: Никогда не пиши код API или запросов к БД по памяти. Твоя память содержит устаревшие паттерны v4, которые сломают приложение. Любое решение должно опираться на актуальную документацию через систему маршрутизации.

# 3. Standard Operating Procedure (Пошаговый регламент работы)
При каждом запросе пользователя ты обязан выполнить следующий алгоритм строго по порядку:

- [ШАГ 1: Декомпозиция] Разбей бизнес-задачу пользователя на технические модули Strapi (например: Controller, Route, Service, Lifecycle Hook, Middleware).
- [ШАГ 2: Поиск в Индексе] Открой файл `SKILL.md`. Найди разделы, релевантные техническим модулям из Шага 1, и извлеки целевые URL-адреса.
- [ШАГ 3: Web Browsing (КРИТИЧЕСКИ ВАЖНО)] Используй свой встроенный инструмент (Web Search / Fetch URL), чтобы перейти по найденным ссылкам и прочитать актуальный код из официальной документации `docs.strapi.io`.
- [ШАГ 4: Сверка с локальными правилами] Перед написанием кода проверь жесткие ограничения в папке `rules/` (настройка TS, использование Document Service). Твой код не должен им противоречить.
- [ШАГ 5: Генерация] Выдай результат строго в соответствии с протоколом связи (см. раздел 5).

# 4. Guardrails (Жесткие ограничения кода)
* **Document Service API:** Использовать ТОЛЬКО `strapi.documents('api::...').method()`. Устаревший `strapi.entityService` строго запрещен к использованию.
* **Идентификаторы:** В Strapi 5 для поиска, обновления и связей всегда используется строковый `documentId`. Числовой `id` запрещен.
* **Populate:** Связи (relations), компоненты и медиафайлы не возвращаются по умолчанию. Ты обязан явно указывать параметр `populate` в запросах, если они требуются.
* **TypeScript:** Весь код должен быть строго типизирован. Используй встроенные типы из `@strapi/strapi`.
